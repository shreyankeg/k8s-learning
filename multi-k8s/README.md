# AWS EC2 + RKE2 Kubernetes Infrastructure

Terraform provisions a full AWS network and 6 EC2 instances. Ansible then installs RKE2 on all nodes and forms **3 independent Kubernetes clusters** — one per Availability Zone — each with a dedicated master and slave node.

The inventory is **auto-generated by Terraform**, so no manual IP editing is ever needed — even after a `terraform destroy && apply`.

## Architecture

```
                        ┌──────────────────────────────────────────────────────────┐
                        │                     VPC  10.0.0.0/16                     │
                        │                                                           │
Internet ──► IGW ─────►│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   │
                        │  │  Public-1   │   │  Public-2   │   │  Public-3   │   │
                        │  │ 10.0.1.0/24 │   │ 10.0.2.0/24 │   │ 10.0.3.0/24 │   │
                        │  │             │   │             │   │             │   │
                        │  │  master-1   │   │  master-2   │   │  master-3   │   │
                        │  │  slave-1    │   │  slave-2    │   │  slave-3    │   │
                        │  └──────┬──────┘   └─────────────┘   └─────────────┘   │
                        │         │ NAT                                            │
                        │  ┌──────▼──────┐   ┌─────────────┐   ┌─────────────┐   │
                        │  │  Private-1  │   │  Private-2  │   │  Private-3  │   │
                        │  │10.0.11.0/24 │   │10.0.12.0/24 │   │10.0.13.0/24 │   │
                        │  └─────────────┘   └─────────────┘   └─────────────┘   │
                        └──────────────────────────────────────────────────────────┘

  Cluster-1: master-1 + slave-1  (AZ-a)
  Cluster-2: master-2 + slave-2  (AZ-b)
  Cluster-3: master-3 + slave-3  (AZ-c)
```

### Resources created by Terraform

| Resource | Count | Details |
|---|---|---|
| VPC | 1 | DNS hostnames and DNS support enabled |
| Internet Gateway | 1 | Attached to the VPC |
| Public Subnets | 3 | One per AZ — auto-assign public IP on launch |
| Private Subnets | 3 | One per AZ — outbound internet via NAT Gateway |
| Elastic IP | 1 | Allocated for the NAT Gateway |
| NAT Gateway | 1 | Lives in public-subnet-1, shared by all private subnets |
| Route Table (public) | 1 | `0.0.0.0/0` → Internet Gateway |
| Route Table (private) | 1 | `0.0.0.0/0` → NAT Gateway |
| SSH Key Pair | 1 | RSA-4096, private key saved to `keys/<key_name>.pem` |
| Security Group | 1 | SSH + K8s API externally; all traffic within VPC |
| EC2 Instances | 6 | `master-1/2/3` + `slave-1/2/3`, 2 per public subnet |
| Ansible Inventory | 1 | `ansible/inventory/hosts.ini` written automatically |

### EC2 instance naming and placement

| Instance | Role | Subnet | AZ | K8s Cluster |
|---|---|---|---|---|
| ec2-node-master-1 | RKE2 server | public-1 | AZ-a | cluster-1 |
| ec2-node-master-2 | RKE2 server | public-2 | AZ-b | cluster-2 |
| ec2-node-master-3 | RKE2 server | public-3 | AZ-c | cluster-3 |
| ec2-node-slave-1  | RKE2 agent  | public-1 | AZ-a | cluster-1 |
| ec2-node-slave-2  | RKE2 agent  | public-2 | AZ-b | cluster-2 |
| ec2-node-slave-3  | RKE2 agent  | public-3 | AZ-c | cluster-3 |

## Prerequisites

### Terraform
- [Terraform](https://developer.hashicorp.com/terraform/downloads) >= 1.3.0
- AWS IAM user or role with permissions to manage: VPC, EC2, Key Pairs, Elastic IPs

### Ansible
- Python 3.8+
- Ansible >= 2.12
- `ansible.posix` and `community.general` collections
- `kubectl` (to interact with the clusters after setup)

## Part 1 — Terraform: Provision the Infrastructure

### Step 1 — Configure credentials

```bash
git clone <your-repo-url>
cd k8s-test

cp terraform.tfvars.example terraform.tfvars
```

Edit `terraform.tfvars` with your real values:

```hcl
aws_access_key = "AKIAIOSFODNN7EXAMPLE"
aws_secret_key = "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
aws_region     = "us-east-1"
ami_id         = "ami-0c02fb55956c7d316"   # Amazon Linux 2023, us-east-1
```

### Step 2 — Find the correct AMI for your region

AMI IDs are region-specific. Find the latest Amazon Linux 2023 AMI:

```bash
aws ec2 describe-images \
  --owners amazon \
  --filters "Name=name,Values=al2023-ami-*-x86_64" \
            "Name=state,Values=available" \
  --query "sort_by(Images, &CreationDate)[-1].ImageId" \
  --output text \
  --region us-east-1
```

Or find it in the AWS Console → EC2 → AMIs → Public images.

### Step 3 — Initialize, plan, and apply

```bash
terraform init
terraform plan
terraform apply
```

Type `yes` when prompted. Apply takes approximately **3–5 minutes** — the NAT Gateway is the slowest part.

`terraform apply` automatically writes `ansible/inventory/hosts.ini` with the correct public IPs. No manual editing is needed.

## Part 2 — Ansible: Install RKE2 and Form Clusters

### Step 1 — Install Ansible and required collections

```bash
pip install ansible
ansible-galaxy collection install ansible.posix community.general
```

### Step 2 — Run the main playbook

```bash
cd ansible
ansible-playbook site.yml
```

The playbook runs 3 plays in sequence:

| Play | Targets | What it does |
|---|---|---|
| 1. rke2_common | All 6 nodes | Loads `br_netfilter` + `overlay` kernel modules, sets sysctl params (`ip_forward`, bridge-nf-call), disables swap, stops firewalld |
| 2. rke2_server | master-1/2/3 | Installs `rke2-server`, writes config with `tls-san` (public IP), starts the service, waits for the node token; rotates the API server cert if config changed |
| 3. rke2_agent | slave-1/2/3 | Reads the node token from its paired master, installs `rke2-agent`, connects to master via **private IP** on port 9345, starts the service |

Total run time: **10–15 minutes**.

### Step 3 — Fetch kubeconfigs

```bash
ansible-playbook fetch_kubeconfigs.yml
```

This fetches `/etc/rancher/rke2/rke2.yaml` from each master, replaces the loopback address with the master's public IP, and saves the files locally:

```
ansible/kubeconfigs/master-1.yaml   ← cluster-1
ansible/kubeconfigs/master-2.yaml   ← cluster-2
ansible/kubeconfigs/master-3.yaml   ← cluster-3
```

Because the server cert includes the public IP in its SAN (set via `tls-san`), kubectl connects without any TLS errors.

### Step 4 — Verify all 3 clusters

```bash
kubectl --kubeconfig ansible/kubeconfigs/master-1.yaml get nodes
kubectl --kubeconfig ansible/kubeconfigs/master-2.yaml get nodes
kubectl --kubeconfig ansible/kubeconfigs/master-3.yaml get nodes
```

Expected output for each cluster (2 nodes, both `Ready`):

```
NAME       STATUS   ROLES                  AGE   VERSION
master-1   Ready    control-plane,master   5m    v1.31.x+rke2r1
slave-1    Ready    <none>                 3m    v1.31.x+rke2r1
```

### Working with multiple clusters at once

```bash
export KUBECONFIG=ansible/kubeconfigs/master-1.yaml:ansible/kubeconfigs/master-2.yaml:ansible/kubeconfigs/master-3.yaml

kubectl config get-contexts          # list all 3 clusters
kubectl config use-context default   # switch active cluster
```

## Reprovisioning

After a `terraform destroy && terraform apply`, the full workflow is the same three commands — no manual steps:

```bash
terraform apply                        # recreates infra + rewrites hosts.ini
cd ansible
ansible-playbook site.yml              # installs RKE2 with correct IPs in cert + agent config
ansible-playbook fetch_kubeconfigs.yml # fetches fresh kubeconfigs
```

## SSH Access

The private key is written to `keys/ec2-key.pem` (chmod 0600) automatically by Terraform.

```bash
ssh -i keys/ec2-key.pem ec2-user@<public_ip>
```

The default SSH user for Amazon Linux 2023 is `ec2-user`.

RKE2 binaries and `kubectl` are available on masters at `/var/lib/rancher/rke2/bin/`. A profile script at `/etc/profile.d/rke2.sh` adds this to PATH — re-login or source it manually:

```bash
source /etc/profile.d/rke2.sh
kubectl get nodes
```

> **Security tip:** Restrict SSH to your own IP instead of `0.0.0.0/0`:
> ```hcl
> # terraform.tfvars
> ssh_allowed_cidrs = ["YOUR_PUBLIC_IP/32"]
> ```

## Security Group Rules

| Port / Protocol | Source | Purpose |
|---|---|---|
| 22 TCP | `ssh_allowed_cidrs` | SSH access to all instances |
| 6443 TCP | `ssh_allowed_cidrs` | Kubernetes API server (`kubectl`) |
| All traffic | VPC CIDR `10.0.0.0/16` | Inter-node: RKE2 join (9345), kubelet (10250), etcd (2379-2380), Canal CNI (8472 UDP), etc. |
| All traffic | — (egress) | Unrestricted outbound internet |

Inter-node ports (9345, 10250, 8472, etc.) are covered by the intra-VPC rule. Agents always connect to masters using **private IPs**, so this traffic never leaves the VPC.

## Tear Down

```bash
terraform destroy
```

Destroys all AWS resources: VPC, subnets, route tables, IGW, NAT Gateway, Elastic IP, EC2 instances, security group, and the AWS key pair. The local `keys/*.pem` file and `ansible/kubeconfigs/` are **not** removed automatically:

```bash
rm keys/ec2-key.pem
rm -rf ansible/kubeconfigs/
```

---

## Security Notes

- `terraform.tfvars` is gitignored — AWS credentials are never committed.
- `keys/*.pem` is gitignored — private keys are never committed.
- `ansible/kubeconfigs/` contains cluster admin credentials — do not commit these files.
- Private subnets have no inbound exposure; outbound internet goes through the NAT Gateway.
- `ssh_allowed_cidrs` defaults to `0.0.0.0/0` for convenience — restrict it to your IP in any non-throwaway environment.
